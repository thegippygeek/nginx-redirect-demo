## Overview

Replaced v1's single flip-in-place proxy with a **blue-green proxy tier**. A front `switch` nginx is the client's only endpoint (`app.demo.test`); it flips a one-line map in `switch/active-proxy.conf` (`default old`в†’`new`) + `nginx -s reload` to route between two **static** single-upstream proxies вЂ” `proxy-old`в†’`server-old` (`app-old.demo.test`) and `proxy-new`в†’`server-new` (`app-new.demo.test`) вЂ” which are never reconfigured during the demo. One selector edit flips **both** HTTP:9092 and SSH:22 (the map file is shared across the http and stream contexts, D-39). The client never changes what it connects to.

The mechanism is identical to v1 (map flip + reload); the architecture around it is what changed вЂ” and it bought three things v1 couldn't do: **pre-flip validation** of the new stack over both protocols before committing, **instant rollback** by flipping the switch back, and **"the old proxy is never touched"** as a `shasum -a 256` proof rather than a claim.
