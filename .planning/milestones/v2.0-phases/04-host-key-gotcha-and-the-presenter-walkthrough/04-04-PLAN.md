## Phase Goal

**As a** presenter, **I want to** deliberately trigger the SSH host-key failure after the cutover and
fix it live without touching the client, **so that** the audience sees the thing that actually breaks
on migration day and the answer that scales to a real fleet.

**This plan's slice of that goal:** durability and honesty. The script stays true to the rig, and the
one claim that rests on judgement is labelled as judgement.

## Plan-level decisions (Claude's discretion, exercised here)

- **The lint extracts from the document rather than grepping for a hand-maintained list.** A list of
  expected commands maintained beside the document is a second source of truth that drifts from the
  first. Pull the commands and paths out of the document itself and check each against the repository.
- **`section_walkthrough` runs before `section_hostkey` in the `all` chain.** It is non-destructive
  and fast, and the destructive section must stay last for the reason already documented there.
- **The cold read is a real `checkpoint:human-verify` task, not a `<human-check>` line inside an
  automated one.** `04-VALIDATION.md:70` prescribes a checkpoint task, and the distinction is
  load-bearing rather than pedantic: an autonomous executor can satisfy an `<automated>` line, mark
  the task done and never halt, so a verify-block-only implementation would leave the specified gate
  existing on paper and nowhere else. This plan is therefore `autonomous: false`.
- **The check is written as a protocol, not a request for approval.** "Read it and say if it looks
  good" produces a yes. "Run it top to bottom on a clean checkout and report every point at which you
  had to guess" produces the information criterion 5 actually needs.
- **The cleanliness gate is scoped to the demo's shipped source, and the VALIDATION edit is committed
  before it runs.** Task 2 mutates `04-VALIDATION.md`, so a whole-tree cleanliness assertion would be
  unsatisfiable by construction — it would see the task's own uncommitted edit. Sequencing the commit
  first and scoping the check to `scripts/`, `Makefile`, `compose.yaml`, `proxy/`, `backend/`,
  `client/`, `status/`, `README.md` and `WALKTHROUGH.md` asserts the property actually worth asserting:
  the phase left no stray edit in the rig's source.
