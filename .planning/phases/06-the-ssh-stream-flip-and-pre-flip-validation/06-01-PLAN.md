Extend the switch's single one-line selector to govern the SSH:22 `stream` path as well as
HTTP:9092, so one edit flips both protocols. This is a verbatim re-home of v1's proven stream
block (`proxy/nginx.conf` lines 201-293) onto `switch/nginx.conf` with exactly three string
edits, followed by reconciling the two deferred SSH test sections (`section_ssh`,
`section_hostkey`) from the v1 `proxy` topology onto the switch and re-enabling them in the
`make test` runner. **This is the Slice A demoable milestone:** one selector edit, both
protocols flip.

**Phase user story (MVP):** **As a** presenter, **I want to** flip the switch selector once and
have the identical `ssh app.demo.test` command land on OLD then NEW just as `curl :9092` does,
**so that** the cutover is a single honest action across both protocols with nothing to change
on the client side.

Purpose: SW-03 is the milestone's mechanism completed — the same `map` file drives http and
stream because an nginx `map` is valid in both contexts while an `upstream` is not (D-13/D-39,
already cashed in the http block). Output: the switch's :22 listener live, and `make test` green
with the whole SSH surface re-enabled. This also finalizes SW-01 (the switch's SSH:22 endpoint).

assumption-delta / api-coverage / schema-gate: N/A — config + POSIX-sh harness; no external API,
no ORM/schema, no new pluralization (this completes the existing old/new selector over a second
protocol). No package installs (no npm/pip/cargo surface).
