## Phase Goal

**As a** presenter on stage, **I want to** flip the nginx upstream from old to new and reload, **so that** the audience sees the identical URL now answered by `server-new`, with independent evidence confirming it.

> Derived mechanically from the ROADMAP `**Goal:**` line for Phase 2, which is written in
> outcome form rather than user-story form. See `## Planner Note` at the foot of this plan.

builds the whole flip pipeline (gate -> rewrite -> validate -> reload -> prove -> settle), the
dual evidence log that every later plan reads, and the unpublished :8081 listener that serves as
proxy healthcheck, reload oracle, and liveness probe.

builds on the evidence log this plan creates.

`clear-evidence`; `section_cutover()` in the smoke suite; README flip narration.

<assumption_delta_decision>
**Signal:** `pluralization` — a second evidence sink is introduced where there was one
(`access_log /dev/stdout demo;`), and the `active-backend.conf` map may gain entries beyond
`default` in Phase 3.

**Primary noun:** *the evidence sink*.

**Decision (sink):** `add-alongside`, deliberately. D-31 locks the stdout `demo` format as the
default and two Phase 1 assertions grep it; the second sink is a purpose-built machine format for
a different consumer with a different lifetime. Promoting a single canonical JSON sink and
rendering the human view from it would break the D-31 "this is what you'd actually see in
production" claim and the Phase 1 assertions simultaneously. **What would force a later promote:**
a third consumer needing yet another shape, or any requirement to change the stdout format.

**Decision (map entries):** `no-change` for Phase 2. The status service parses `default` only and
reports any additional non-`default` map entry in its JSON under `extra_map_entries` so a Phase 3
port-keyed override surfaces rather than being silently ignored (RESEARCH Open Question 4,
resolved here).

**Suggested invariant test (accepted):** `section_cutover()` asserts proxy/active-backend.conf is
still exactly five lines with both `^#` comment lines after a flip in either direction. That goes
red the instant a future phase reintroduces a structure the presenter cannot read in full on
screen.
</assumption_delta_decision>
